
# Мини-оболочка с файловыми командами на Python

Простая командная оболочка на Python с набором файловых операций.
Изначально текущая директория: '~'

### Установка
Клонируйте репозиторий \
git clone <[ссылка_на_репозиторий](https://github.com/jurolli/Lab_2/tree/main/src)>


### Используйте pip
pip install -r requirements.txt #


### Базовые операции

- `ls [path] [-l]` - просмотр содержимого директории
- `cd [path]` - смена директории (поддерживает `..`, `~`)
- `cat [file]` - просмотр содержимого файла
- `cp [source] [destination] [-r]` - копирование файлов/директорий
- `mv [source] [destination]` - перемещение/переименование
- `rm [file] [-r]` - удаление файлов/директорий

### Реализованные команды:

### cd:
```
cd <path>
```
Описание: Изменяет текущую рабочую директорию \
Аргументы: \
path - путь к целевой директории. Если не указан, перемещает в домашнюю директорию (~) \
Поддерживает специальные символы: .. (родительская директория), ~ (домашняя директория)


### ls:
```
ls -[l]
```
Описание: Отображает содержимое директорий \
Аргументы: \
-l  - подробный вывод с дополнительной информацией


### cat:
```
cat <file>
```
Описание: Выводит содержимое текстового файла \
Аргументы: \
file  - путь к файлу для чтения

### cp
```
cp <what> <where> -[r]
```
Описание: Копирует файлы или директории \
Аргументы: \
what  - исходный путь \
where - целевой путь \
-r - рекурсивное копирование директорий

### mv
```
mv <what> <where>
```
Описание: Перемещает или переименовывает файлы/директории \
Аргументы: \
what - исходный путь \
where  - целевой путь

### rm
```
rm <path> -[r]
```
Описание: Безопасно удаляет файлы или директории, перемещая их в папку .trash \
Аргументы: \
path - путь к удаляемому объекту \
-r  - рекурсивное удаление директорий

### Особенности реализации
__Безопасность__
- Защита от удаления системных директорий (/, ..)
- Подтверждение перед рекурсивным удалением директорий
- Все удаленные файлы сохраняются в .trash для возможности восстановления

__Oбработка ошибок__
- Все команды включают обработку исключений
- Детальные сообщения об ошибках для пользователя
- Логирование всех операций с указанием успешности

__Логирование__
- Модуль log.py записывает все выполненные команды с указанием:
- - Времени выполнения
- - Успешности операции
- - Сообщений об ошибках (если есть)

__Управление корзиной__
- При удалении файлов:
- - Создается папка .trash (если не существует)
- - Удаляемые файлы перемещаются в .trash
- - При конфликте имен добавляется временная метка
- - Файлы физически не удаляются с диска
